# 内容上传 Skill

## 四个兴趣小组

| 小组名 | 别名/简称 |
|--------|----------|
| 财富管理 | 财富、理财、wealth |
| 金融分析 | 金融、finance |
| 内容创作 | 内容、创作、content |
| 其他 | other |

## 触发方式

用户输入类似以下内容时激活：
- `/内容上传`
- "帮我上传这个到社区"
- "把这个加到对应目录"
- "整理这个内容到 community"

**指定小组的写法（任意一种均可）：**
- `/内容上传 财富管理`
- `/内容上传 小组：金融分析`
- "上传到内容创作小组"
- "放到财富管理的 wiki"

## 目录结构

```
community/
├── 财富管理/
│   ├── skills/     # Claude Code skill 文件（.md，可直接 /install）

│   ├── wiki/       # 知识库条目（.md，长篇参考）
│   └── 笔记/       # 会议记录、idea、心得（.md）
├── 金融分析/
│   ├── skills/
│   ├── prompts/
│   ├── wiki/

├── 内容创作/

│   ├── prompts/

└── 其他/
    ├── skills/
    ├── prompts/
    ├── wiki/
    └── 笔记/
```

## 执行步骤

### Step 1：获取内容

支持三种输入：

**A. 直接粘贴文字**
用户直接贴出内容，读取全文。

**B. 本地文件路径**
用 Read 工具读取文件内容。

**C. GitHub 项目 URL**
用 WebFetch 获取 README 和主要文件内容。如果是 skill 文件（含 frontmatter `---`），直接使用原始内容；否则提取核心内容整理为 Markdown。

### Step 2：判断分类

**如果用户已指定小组，直接使用，跳过自动判断。**
别名映射：财富/wealth → 财富管理；金融/finance → 金融分析；内容/创作/content → 内容创作；其他/other → 其他。

否则根据内容关键词自动判断：

**小组判断：**
| 小组 | 关键词 |
|------|--------|
| 财富管理 | 理财、税务、保险、退休、401k、RRSP、房产、遗产、北美、加拿大、美国税、报税 |
| 金融分析 | 股票、基金、ETF、期权、市场分析、财报、估值、技术分析、量化、数据 |
| 内容创作 | 公众号、小红书、视频、脚本、PPT、标题、选题、封面、文章、写作 |
| 其他 | 职场、邮件、报告、会议、效率、日常、自动化（不属于以上三类时） |

**类型判断：**
| 类型 | 判断标准 |
|------|---------|
| skills | 含 frontmatter（`---` 开头）且有 `name:` 字段；或内容是可直接用于 Claude Code 的 skill 指令文件 |
| prompts | 含明确的 prompt 模板、变量占位符（如 `{{变量}}`）、或 "你是一个..." 开头的系统 prompt |
| wiki | 长篇解释性内容、概念介绍、操作指南、知识梳理 |
| 笔记 | 会议记录、个人 idea、心得体会、讨论摘要 |

**不确定时：** 向用户展示判断结果并询问确认，不要直接写文件。

### Step 3：确定文件名

- 用简短中文或英文描述内容主题
- 格式：`主题名称.md`（如 `美股税务申报指南.md`、`xhs-title-generator.md`）
- 如果是 skill 文件，文件名用 skill 的 `name:` 字段
- 避免特殊字符，空格用 `-` 替代

### Step 4：写入文件

目标路径：`{小组}/{类型}/{文件名}.md`

**写入前展示确认信息：**
```
准备写入：
- 小组：财富管理
- 类型：wiki
- 路径：财富管理/wiki/美股税务申报指南.md
- 内容预览：（前 3 行）

确认？(y/n)
```

用户确认后再写入。

### Step 5：更新小组 README（index）

读取 `{小组}/README.md`，找到或创建 `## 内容列表` 区块，在对应类型小节下追加一行：

```
- [{标题}]({相对路径}) — {一句话描述} `{今天日期}`
```

**README 内容列表结构：**
```markdown
## 内容列表

### skills
- [AI 投资委员会](skills/investment-committee/README.md) — 7人格并行分析股票 `2026-07-15`

### prompts
（暂无）

### wiki
（暂无）

### 笔记
- [LibTV Agent 视频工作流](笔记/LibTV-Agent-视频工作流.md) — 100+ AI视频Skill，自动生成分镜 `2026-07-15`
```

规则：
- 如果该类型小节不存在，新建它
- 如果该类型小节存在但只有「（暂无）」，替换掉那行
- 一句话描述从内容的第一段/标题摘取，≤20 字
- skill 目录型内容（多文件项目）链接到其 README.md，单文件链接到文件本身

### Step 6：完成提示

写入内容文件 + 更新 README 后告知用户：

```
已写入：财富管理/wiki/美股税务申报指南.md
已更新：财富管理/README.md（内容列表）

需要提交到 GitHub 吗？说"帮我提交并推送"，我来处理。
```

如果用户说"提交"/"push"/"推送"或类似指令，执行：
1. `git add {写入的文件路径} {小组}/README.md`
2. `git commit -m "docs: 添加{内容主题}"`（commit message 用中文描述内容）
3. `git push`

并向用户报告结果。

## 特殊情况处理

**GitHub skill 项目**
如果用户提供 GitHub URL 且项目包含多个 skill 文件，询问用户要导入哪个文件，或全部导入。

**内容跨多个方向**
如果内容同时适合多个方向，询问用户选择，或分别写入多个目录（需用户确认）。

**文件已存在**
如果目标文件已存在，展示现有内容和新内容的差异，询问是覆盖还是追加。

**内容需要整理**
如果原始内容格式混乱（如聊天记录截图描述、口述内容），先整理成规范 Markdown 再写入，整理后让用户确认。
